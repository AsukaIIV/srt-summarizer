# SRT-SUMMARIZER

### 把课堂字幕、转录文本和课堂视频整理成适合复习的 Markdown 笔记

---

## 软件简介

SRT-SUMMARIZER 是一个面向学生和日常学习场景的 Windows 桌面工具。

它可以把课堂字幕、转录文本、视频截图、往期笔记整理成一份结构清晰、方便复习、适合长期保存的 Markdown 课程笔记。

它不是聊天工具，而是专门用来做 **课堂资料整理** 的软件。

---

## 适合谁使用

- 想把 `.srt` / `.txt` / `.md` 快速整理成课程笔记的学生
- 有录播课程，需要边看边沉淀笔记的人
- 想把“字幕 + 视频 + 往期笔记”整合成一份复习资料的人
- 想长期积累整门课 Markdown 笔记的人

---

## 主要功能

- 支持整理 `.srt` / `.txt` / `.md` 为结构化课程笔记
- 支持配对视频并自动抽取课堂截图
- 支持图文混排，帮助理解公式、板书、图示、例题过程
- 支持批量扫描资料目录
- 支持导入往期笔记，增强课程连续性
- 支持保存 API 配置，下次启动自动恢复
- 支持按平台分别记住模型名称、接口地址和 API Key
- 支持启动时自动校验当前 API 配置
- 支持实时查看生成进度和模型输出
- 支持输出到固定目录或源文件所在目录
- 支持生成结构化图示，增强考前复习效果

当前内置平台：

- `DeepSeek`
- `OpenAI Compatible`
- `硅基流动`
- `千问`
- `Kimi`
- `Claude`

其中：

- `DeepSeek`、`OpenAI Compatible`、`硅基流动`、`千问`、`Kimi` 默认走 OpenAI 兼容接口
- `Claude` 默认走 Anthropic Messages 原生接口

---

## 使用前需要准备

在开始之前，你通常需要准备：

- 一份字幕或转录文件（`.srt` / `.txt` / `.md`）
- 如果要生成图文混排结果，还需要对应视频文件
- 一个可用的 API 平台账号
- API Key
- 对应平台的模型名称和接口地址

---

## API 获取说明

本软件本身 **不提供 API**，你需要从自己使用的平台申请。

通用流程通常是：

1. 注册模型平台账号
2. 打开该平台的开发者/API 页面
3. 创建或复制 API Key
4. 查找接口地址（Base URL）
5. 查找可用模型名称
6. 回到本软件，在 **API设置** 页面填写并保存

需要填写的内容通常包括：

- API 平台
- 模型名称
- 接口地址
- API Key

安全建议：

- 不要把 API Key 发给别人
- 不要把 API Key 上传到公共仓库
- 如果怀疑泄露，请及时去平台后台更换

常用平台参考：

- DeepSeek 官网：<https://platform.deepseek.com/>
- Anthropic 官网：<https://console.anthropic.com/>
- 硅基流动官网：<https://cloud.siliconflow.cn/>
- 阿里云百炼 / 千问 API：<https://bailian.console.aliyun.com/>
- Moonshot / Kimi 开放平台：<https://platform.moonshot.cn/>

## 使用方法（仅 Windows `.exe`）

### 1. 启动程序

双击运行：

```text
SRT-SUMMARIZER-v1.0.1.exe
```

你不需要额外安装 Python。

---

### 2. 配置 API

首次使用时，在 **API设置** 页填写并保存：

- API 平台
- 模型名称
- 接口地址
- API Key

程序启动时会自动校验当前配置：

- 配置可用：直接正常使用
- 配置缺失或不可用：会自动跳转到 **API设置** 页并显示提示

补充说明：

- 当前平台会分别记住自己的模型名称、接口地址和 API Key
- 切换平台时，会恢复你上次保存或填写过的该平台配置
- 如果你没有手动改过模型或接口地址，切换平台时会自动带入该平台默认值

---

### 3. 选择资料

在 **选择文件** 页中，你可以：

- 选择字幕文件
- 选择视频文件
- 扫描资料目录
- 选择往期笔记

如果希望生成图文混排结果，需要让字幕和对应视频成功配对。

---

### 4. 补充课程信息

在 **课程设置** 页中填写：

- 课程名称
- 课程总体要求（可选）

---

### 5. 开始整理

切换到 **开始运行** 页，点击 **开始整理**。

程序会开始生成课程笔记，并在界面中实时显示：

- 当前进度
- 当前任务状态
- 模型输出内容

---

## 输出结果说明

每个任务通常会生成一个独立课程目录，例如：

```text
某节课_来源目录_课程名/
├── 某节课_课程名_课堂总结.md
└── imgs/
    ├── 001.png
    ├── 002.png
    └── ...
```

其中：

- `课堂总结.md`：最终整理结果
- `imgs/`：课堂截图和程序生成的结构化图示
- 纯字幕任务：通常不会生成截图

Markdown 文件头会包含：

- 生成时间
- 源文件路径
- 模型平台
- 模型名称

---

## 图文混排说明

当字幕成功配对到视频后，任务会进入图文混排模式：

- 程序会尝试自动提取课堂截图
- 截图会插入到更适合阅读和理解的位置附近
- 更适合用于定义、推导、公式变形、图示和例题步骤
- 如果视频任务没有成功提取到有效截图，该任务会直接失败
- 不会自动降级成纯文本模式

---

## 结构化图示说明

在适合的课程内容中，程序还会尝试生成结构化图示，用于强化考前复习，例如：

- 概念/条件对比图
- 答题步骤流程图
- 核心公式关系图

这些图示会和课堂截图一起保存到 `imgs/` 目录，并追加到最终 Markdown 中。

---

## 配置保存位置

程序会自动把配置保存到：

```text
~/.srt_summarizer/settings.json
~/.srt_summarizer/secrets.json
```

下次启动时会自动读取。

## 常见问题排查

如果使用过程中出现问题，可以优先检查：

- API Key 是否正确
- 接口地址是否正确
- 模型名称是否正确
- 当前选择的平台是否和 Key / URL / 模型对应
- 字幕文件是否成功导入
- 视频文件是否与字幕成功配对
- 输出目录是否可写

如果是视频任务失败，还可以检查：

- 视频文件是否正常
- 是否成功提取到有效截图

如果是 Claude 无法连通，还可以检查：

- 是否填写了 Anthropic 可用的 API Key
- 接口地址是否仍为 `https://api.anthropic.com/v1/messages`
- 模型名称是否为 Claude 可用模型

---

## 提示

如果你是第一次使用，建议先用一节较短课程做测试，确认：

- API 配置可用
- 能正常生成 Markdown
- 如果有视频，能正常生成 `imgs/`

确认无误后，再批量整理整门课资料。
